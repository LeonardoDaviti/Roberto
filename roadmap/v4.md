# Roberto v4 Roadmap

## Objective
Improve output quality with measurable, repeatable controls.

## Scope

### 1) Embeddings Retrieval (Context Continuity)
- Add deterministic embedding index for tweets and stories.
- Retrieve top-k prior context per user before user-note generation.
- Retrieve top-k prior story context before digest generation.
- Feed retrieval context into LLM prompts as structured JSON.

### 2) Evaluation Command
- Add `python -m roberto_app.cli eval`.
- Evaluate on fixed fixture data for deterministic metrics.
- Metrics:
  - `citation_coverage`
  - `invalid_citation_rate`
  - `duplicate_notecard_rate`
  - `note_churn`
  - `story_continuity_score`
- Return non-zero exit code on threshold failures.

### 3) Configurable Thresholds
- Add `v4.eval.thresholds` and retrieval config in `settings.yaml`.
- Keep defaults strict but practical.

## Definition Of Done
- v1/v2 pipelines include retrieval context in LLM calls.
- `eval` prints metrics and pass/fail result.
- Tests verify retrieval and evaluator determinism.

## Out Of Scope (v5)
- External vector DB backend.
- Human feedback UI.
- Multi-model ensemble evaluation.

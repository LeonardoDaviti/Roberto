# Roberto v2 Roadmap

## Scope
`python -m roberto_app.cli v2` performs incremental updates:
1. Load followed usernames and cached `last_seen_tweet_id`.
2. Pull only new posts using `since_id` with retries/backoff.
3. Skip LLM + note rewrite when no new tweets for a user.
4. For users with new tweets, rebuild auto block from latest 100 cached tweets.
5. Update user `last_seen_tweet_id` and poll timestamps.
6. Always create a new digest note each run.
7. Export run report JSON listing `created_notes`, `updated_notes`, and per-user new tweet counts.

## Definition Of Done
- Only users with fresh posts get user-note rewrites.
- No duplicate tweets after repeated incremental runs.
- Digest note is always newly created each run.
- Report export differentiates created vs updated notes.

## Extension Hooks
- Topic/story notes that persist and evolve across runs.
- Entity and thread linking across users.
- Embeddings for semantic retrieval and cross-run clustering.

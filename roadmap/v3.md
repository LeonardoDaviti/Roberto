# Roberto v3 Roadmap

## Goals
v3 turns Roberto into a resilient, modular knowledge pipeline:
1. Source-agnostic ingestion contract.
2. Persistent story memory across runs.
3. Strict citation validation before writing notes.
4. Operational CLI split: `sync` (ingest), `build` (notes), `stories status` (memory visibility).
5. Safety/reliability improvements: JSON status output and single-run lock.

## Scope

### 1) Canonical Source Layer
- Add `CanonicalPost` model for all ingest paths.
- Add provider protocol (`SourceProvider`) for future X/local providers.
- Keep storage and note generation independent from source-specific response shapes.

### 2) Story Memory
- Add DB tables:
  - `stories`
  - `story_sources`
- Persist digest stories each run by deterministic slug.
- Track `first_seen_run_id`, `last_seen_run_id`, `mention_count`, `confidence`, tags.
- Create/update story notes at `notes/stories/<slug>.md` with marker-only updates.

### 3) Citation Validation Gate
- Validate per-user and digest blocks against available tweet IDs before rendering.
- Drop unsupported claims/connections when source IDs are missing from cache.
- Ensure all published claims remain traceable to stored tweet IDs.

### 4) CLI and Ops
- `status --json`
- `sync` command (ingest only)
- `build` command (DB-to-notes generation; no X fetch)
- `stories status [--json]`
- Add run lock file to avoid concurrent mutation runs.

## Definition Of Done
- Story notes persist and evolve across multiple runs.
- Invalid citations are filtered before note writes.
- `sync` and `build` work independently.
- `status --json` and `stories status --json` provide machine-readable ops output.
- Existing `v1` and `v2` remain available.

## Out Of Scope (v4 candidates)
- Embeddings-backed retrieval and similarity clustering.
- Automated quality scoring dashboards.
- Multi-step evaluator with benchmark suite.

# Roberto v26 - Book Reading Mode

## Objective
Add a first-class book reading workflow that ingests local files (PDF-first), distills Greene-style notecards with citations, and writes durable notes to `Books/`.

## Scope

### 1) Book Ingestion
- New CLI command: `python -m roberto_app.cli book <path>`
- Supported formats:
  - `.pdf` (via `pypdf`)
  - `.txt`
  - `.md`
- Chunking:
  - Split extracted text into bounded chunks (`v26.chunk_chars`)
  - Keep page range metadata for each chunk
  - Cap chunk count (`v26.max_chunks_per_book`) to control cost

### 2) SourceRef-First Evidence
- For each chunk, persist canonical `SourceRef` + `SourceSnapshot`:
  - `provider=book`
  - `source_id=book:<file_hash>`
  - `anchor_type=chunk`
  - `anchor=chunk:<idx>:p<start>-<end>`
- LLM cards must cite `source_refs` from allowed chunk references.

### 3) LLM Distillation
- Per chunk call Gemini JSON mode with `BookChunkAutoBlock` schema.
- Extract:
  - `chunk_summary`
  - `themes`
  - Greene-style `notecards` (claim/evidence/angle/principle + strategic use + reusable quote + tags + source_refs)

### 4) Notes Output
- Write/update one note per book in:
  - `Books/<book-slug>.md`
- Preserve manual content via marker-only auto block updates.
- Index note in `note_index` with `note_type=book`.

### 5) Token Usage Visibility
- Persist one row per LLM query in `llm_query_usage`:
  - `query_kind`, `query_ref`, `model`
  - `prompt_tokens`, `output_tokens`, `total_tokens`
  - `cached` flag
  - `run_id`
- CLI prints per-query token table and total usage.
- Export full run payload with token usage to:
  - `data/exports/book_<run_id>.json`

## Config (`config/settings.yaml`)
```yaml
v26:
  enabled: true
  books_dir: Books
  chunk_chars: 4500
  max_chunks_per_book: 80
  cards_per_chunk: 6
```

## Acceptance Criteria
- Running `book <path_to_pdf>` creates or updates `Books/<slug>.md`.
- Output cards are citation-backed using canonical `source_refs`.
- Per-query token usage is visible in CLI and persisted in DB/export JSON.
- Existing v1/v2 flows continue working unchanged.

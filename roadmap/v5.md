# Roberto v5 Roadmap

## Objective
Deliver appliance-level comfort and reliability: recoverable runs, deterministic state transitions, and actionable operator diagnostics.

## Scope

### 1) Reliability Kernel
- Add per-run structured journal (`data/logs/journal_<mode>_<run_id>.jsonl`).
- Add checkpoint state (`data/logs/checkpoint_<mode>.json`) with completed/failed users.
- Add resume support (`v1 --resume`, `v2 --resume`) to continue from checkpoint.
- Add per-user DB transactions to avoid partial writes on mid-user failures.
- Add atomic note writes via temp-file + rename.

### 2) Doctor Command
- Add `python -m roberto_app.cli doctor` with optional `--online` and `--json`.
- Validate:
  - required files/config
  - env keys
  - timezone validity
  - DB schema/tables
  - lock state
  - writable directories
  - disk free space
  - last run visibility
- Online mode validates X API connectivity/authorization state with clear classification.

## Definition Of Done
- Interrupted v1/v2 runs can resume and skip already completed users.
- Journal/checkpoint files are produced for mutating runs.
- Notes are written atomically.
- `doctor` returns actionable findings and non-zero exit code on blocking issues.
- Regression tests cover reliability and doctor flows.

## Out Of Scope (v6)
- Approval workflow / staged publish gate.
- Rollback snapshots + restore command.
- daemon mode and outbound notifications.

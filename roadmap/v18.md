# Roberto v18 Roadmap

## Objective
Deliver a daily briefing that gives maximum signal in minimum time.

## Scope

### 1) One-Page Daily Brief
- Generate `notes/briefings/<YYYY-MM-DD>.md` each run-day with:
  - Top 5 story deltas.
  - Top 3 cross-story connections.
  - Top 3 idea cards.
- Keep strict citation links for every brief item.

### 2) Change-Based Ranking
- Rank by change since prior run, not static importance:
  - new evidence volume
  - confidence delta magnitude
  - contradiction status changes
  - attention controls (pin/mute/snooze from v15)

### 3) Read Modes
- CLI:
  - `python -m roberto_app.cli brief --mode fast`
  - `python -m roberto_app.cli brief --mode deep`
- Fast mode: compact bullets with citations.
- Deep mode: expands evidence, conflicts, and claim ledger snippets.

## Data Model
- `briefings` table storing generated briefing metadata and run linkage.
- Optional `briefing_items` table for item-level explainability and ranking features.

## Acceptance Criteria
- Daily brief renders in under 2 seconds from local cache.
- Brief includes only cited items.
- Fast mode is readable in <3 minutes; deep mode supports drill-down.

## Tests
- Ranking determinism tests on fixture runs.
- Citation completeness tests.
- Fast vs deep rendering tests.

## Out Of Scope (v19)
- Email/slack notifications.
- Personalized multi-user feeds.

# Roberto v19 Roadmap

## Objective
Ship Greene Mode core loop: capture -> distill -> winnow keeper cards with strict citations.

## Scope

### 1) Multi-pass Distillation
- Pass A (`captured`): generate literal claim/evidence/angle cards from story memory + claim ledger.
- Pass B (`distilled`): enrich every card with:
  - `theme`
  - `principle`
  - `strategic_use_case`
  - `reusable_quote`
- All cards must retain source refs (`username`, `tweet_id`).

### 2) Keeper Winnowing
- Weekly winnow by `(story_id, week_key)`.
- Configurable cap: `v19.keeper_cap_per_story_week`.
- Overflow state controlled by `v19.auto_reject_overflow`.

### 3) CLI + Storage
- `greene sync` runs capture/distill/winnow.
- `greene cards` lists card inventory with state filters.
- SQLite table: `greene_cards`.

## Acceptance Criteria
- Keeper deck remains citation-backed.
- Card volume is bounded by weekly cap.
- Re-running cycle is deterministic for same inputs.

# Roberto v15 Roadmap

## Objective
Enable deliberate human curation of story structure and attention.

## Scope

### 1) Story Surgery
- Add CLI:
  - `python -m roberto_app.cli stories merge <story_a> <story_b> --into <target_slug>`
  - `python -m roberto_app.cli stories split <story_slug> --plan <json_file>`
- Merge behavior:
  - preserve all sources/history.
  - retain deterministic canonical story ID.
  - create alias redirects from old slugs.
- Split behavior:
  - produce multiple child stories with preserved evidence subsets.
  - maintain lineage metadata (`parent_story_id`).

### 2) Attention Controls
- Add status controls for stories/entities:
  - `pinned`
  - `muted`
  - `snoozed_until`
- CLI:
  - `stories pin|mute|snooze|unmute|unpin`
  - `entity pin|mute|snooze|unmute|unpin`
- Ranking pipelines must respect attention state.

### 3) Taxonomy Governance
- Tag dictionary file: `config/taxonomy.yaml`.
- Entity alias overrides file: `config/entity_alias_overrides.yaml`.
- Deterministic override precedence:
  - manual override > inferred alias > heuristic extraction.

## Data Model
- `story_aliases` table (`alias_slug -> story_id`).
- `story_lineage` table (`parent_story_id -> child_story_id`).
- `attention_state` table for stories/entities.
- `taxonomy_terms` optional cache table if needed.

## Acceptance Criteria
- Story merge/split operations are reversible and preserve source links.
- Pin/mute/snooze changes affect briefing and search ordering immediately.
- Manual alias/taxonomy overrides remain stable across runs.

## Tests
- Merge/split integrity and lineage tests.
- Attention-state ranking tests.
- Alias override precedence tests.

## Out Of Scope (v16)
- Collaborative multi-user taxonomy editing UI.

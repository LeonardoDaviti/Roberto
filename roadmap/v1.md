# Roberto v1 Roadmap

## Scope
`python -m roberto_app.cli v1` performs an initial deterministic build:
1. Load usernames from `config/following.txt`.
2. Resolve each username to X user ID.
3. Pull latest 100 posts per user via official X API (`exclude=replies,retweets`).
4. Cache users + tweets in SQLite (`data/roberto.db`) with dedupe by tweet ID.
5. Generate per-user auto blocks with Gemini JSON mode.
6. Create/update `notes/users/<username>.md` with marker-only overwrite.
7. Create one per-run digest note in `notes/digests/`.
8. Persist run stats and a machine-readable report in `data/exports/`.

## Definition Of Done
- Running v1 twice does not duplicate tweets.
- Running v1 twice preserves manual note content outside markers.
- A user note exists for every followed username.
- Digest note is created for each v1 run.
- Run metadata is stored in `runs` and report JSON is exported.

## Known Limitations
- No story thread persistence yet.
- No entity/topic memory layer yet.
- Only user notes and run-level digest notes are generated.

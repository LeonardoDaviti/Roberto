# Roberto v16 Roadmap

## Objective
Make uncertainty explicit: confidence, contradictions, and claims become first-class tracked objects.

## Scope

### 1) Conflict Nodes
- Promote contradictions from plain notes into structured graph objects.
- Add conflict object with:
  - `conflict_id`
  - `topic`
  - `claim_a`, `claim_b`
  - supporting source refs
  - open/closed status
- Never auto-merge contradictory claims.

### 2) Confidence Evolution
- Track confidence over time per story/entity:
  - previous confidence
  - new confidence
  - short reason string (deterministic template + evidence counts)
- Render confidence timeline in story/entity notes.

### 3) Claim Ledger
- For each story, maintain compact claim ledger:
  - claim text
  - evidence tweet IDs
  - confidence
  - status (`active|retracted|contested`)

## Data Model
- `conflicts` table (first-class conflicts).
- `confidence_events` table.
- `story_claims` table.
- Links from conflicts/claims to stories/entities and source refs.

## Acceptance Criteria
- Conflicting evidence appears as explicit conflict records.
- Story confidence can be traced across runs.
- Claim ledger remains source-backed and queryable.

## Tests
- Conflict detection determinism tests.
- Confidence transition rule tests.
- Claim status lifecycle tests.

## Out Of Scope (v17)
- Bayesian confidence model.
- Human voting workflow.

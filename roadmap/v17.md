# Roberto v17 Roadmap

## Objective
Version prompts/schemas as shippable assets with regression-safe deployment gates.

## Scope

### 1) Prompt + Schema Registry
- Add folder layout:
  - `prompts/vNN/user_summary.md`
  - `prompts/vNN/digest.md`
  - `prompts/vNN/cards.md`
  - `schemas/vNN/*.json`
- Each run stamps versions in run metadata:
  - `prompt_pack_version`
  - `schema_pack_version`

### 2) Golden Fixtures
- Expand fixture suite with curated, labeled cases:
  - entity alias edge cases
  - contradiction-heavy runs
  - sparse/noisy timelines
  - high-churn runs
- Add expected-output checks for key structures.

### 3) Regression Gates
- Extend `eval` command to compare against baseline runs.
- Fail CI when guarded metrics regress:
  - citation coverage
  - invalid citation rate
  - story continuity
  - note churn
  - contradiction precision

## Data Model
- Optional `model_registry` / `prompt_registry` run metadata tables.
- Existing `runs.stats_json` includes version and gate results.

## Acceptance Criteria
- Changing prompts/schemas requires explicit version bump.
- Run exports contain prompt/schema version stamps.
- Regression gates block degraded builds deterministically.

## Tests
- Version resolution tests.
- Fixture compatibility tests across versions.
- Gate threshold fail/pass tests.

## Out Of Scope (v18)
- Auto-tuning prompts in production.

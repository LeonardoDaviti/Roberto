# Roberto v25 Roadmap

## Objective
Make Roberto truly multi-source by replacing tweet-ID-centric citations with provider-agnostic `SourceRef` contracts, while preserving existing reliability, memory, and validation invariants.

## Why v25
- Current citation and validation logic is still centered on X tweet IDs.
- Multi-platform scale (X + blogs + YouTube) requires stable, source-native anchors.
- Existing methodology remains unchanged; only the evidence contract is generalized.

## Scope

### 1) SourceRef Contract (Core)
- Introduce canonical citation object:
  - `provider` (`x|rss|web|youtube|...`)
  - `source_id` (tweet ID, canonical URL, video ID, etc.)
  - `url`
  - `anchor_type` (`id|hash|dom|timecode|chunk`)
  - `anchor` (tweet id, content hash, selector, `HH:MM:SS-HH:MM:SS`, transcript chunk id)
  - `excerpt_hash`
  - `snapshot_hash`
  - `captured_at`
- All downstream objects (stories, cards, conflicts, claims, briefs, drafts) reference `SourceRef[]`.

### 2) Canonical Evidence Layer
- Add durable source snapshot model:
  - normalized text payload
  - metadata payload
  - stable hash for replay and validation
- Keep provider-specific ingestion adapters, but store provider-agnostic canonical evidence.

### 3) Citation Gate v2
- Replace tweet-id-only validation with `SourceRef` validation:
  - ref must resolve to cached snapshot
  - anchor must be valid for provider type
  - excerpt hash must match snapshot segment
- Reject writes when invalid refs exceed threshold.

### 4) Provider Protocol Expansion
- Keep current source protocol shape, but enforce mandatory `to_source_ref()` mapping for every provider.
- X provider remains first-class through compatibility shim (`tweet_id -> SourceRef`).
- Add initial non-X providers:
  - RSS/Web article adapter (URL + excerpt/hash anchors)
  - YouTube adapter (video ID + timecode/transcript chunk anchors)

### 5) Migration Strategy (No Big-Bang Rewrite)
- Phase A: dual-write (`tweet_id` + `SourceRef`) for X.
- Phase B: backfill existing records to `SourceRef`.
- Phase C: flip validators/renderers to `SourceRef` as source of truth.
- Phase D: retain tweet-ID fields only as compatibility aliases.

## Phase Status
- Phase A: implemented (dual-write during tweet ingest + SourceRef stats/validate/backfill CLI)
- Phase B-D: pending

### 6) CLI + Ops
- `sources backfill` (migrate legacy tweet refs to `SourceRef`)
- `sources validate` (audit unresolved/invalid refs)
- `sources stats` (coverage by provider, anchor validity, unresolved refs)
- `sync --provider x|rss|youtube|all`

### 7) Notes and Studio Rendering
- Render citations from `SourceRef` uniformly:
  - X: `@user:id`
  - Web/RSS: domain + title + anchor hash
  - YouTube: video ID + timecode
- Preserve compile-mode behavior while keeping traceability in non-compile outputs.

## Data Model
- Add/upgrade tables:
  - `source_refs` (canonical refs)
  - `source_snapshots` (durable captured evidence)
  - join tables from stories/cards/claims/conflicts/briefing_items/studio_outputs -> refs
- Existing tweet-centric tables remain during migration window.

## Acceptance Criteria
- Roberto runs with mixed providers in a single run.
- Citation gate works identically across X, web, and YouTube.
- `SourceRef` coverage on new outputs is 100%.
- Legacy X-only flows keep working without regression.
- Draft and briefing traceability remains deterministic.

## Test Plan
- Contract tests for each provider `to_source_ref()` mapping.
- Anchor validation tests (`id/hash/dom/timecode/chunk`).
- Migration tests from tweet IDs to `SourceRef`.
- Cross-provider pipeline smoke test (X + web + YouTube mock data).
- Regression gates include:
  - `source_ref_coverage`
  - `invalid_source_ref_rate`
  - `unresolved_source_ref_rate`
  - existing citation/conflict/continuity metrics.

## Out Of Scope (v26)
- Book-native long-text ingestion and location model (page/loc/chapter semantics).
- Provider-specific ranking models.

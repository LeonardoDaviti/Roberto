# Roberto v13 Roadmap

## Objective
Add an editorial control plane that makes automated note generation safe, reviewable, and reversible.

## Problem
As automation expands (stories, entities, cards, digests), trust drops without explicit review and recovery mechanics.

## Scope

### 1) Staging-First Publishing
- Add staging destination: `notes/_staging/<run_id>/...`.
- When `v13.enabled=true`, v1/v2/build write all auto-generated updates to staging paths first.
- Keep live notes unchanged until explicit promotion.
- Keep marker-only overwrite behavior in staged writes.

### 2) Diff-First Review
- Add CLI: `python -m roberto_app.cli editor review --run-id <run_id>`.
- For each staged note/story:
  - show live path, staged path, note type.
  - show unified diff preview.
  - show source refs that triggered this update (`username + tweet_id`).
- Add JSON mode for machine review: `--json`.

### 3) Promote Flow
- Add CLI: `python -m roberto_app.cli editor promote --run-id <run_id>`.
- For each staged item:
  - snapshot current live note before promote.
  - atomically replace live note with staged content.
  - mark staged item as promoted.
- Update run report metadata with approved notes.

### 4) Rollback Flow
- Add CLI:
  - `python -m roberto_app.cli editor rollback --note <path>`
  - `python -m roberto_app.cli editor rollback --note <path> --snapshot-id <id>`
- Restore note/story from snapshots.
- Snapshot current content before rollback.
- Update run report metadata with rolled back notes when applicable.

## Data Model

### New table: `staged_notes`
- `run_id` (PK-part)
- `live_path` (PK-part)
- `staged_path`
- `mode`
- `note_type`
- `trigger_refs_json` (list of `{username,tweet_id}`)
- `status` (`staged|promoted|discarded`)
- `created_at`
- `promoted_at` (nullable)

### New table: `note_snapshots`
- `snapshot_id` (PK autoincrement)
- `note_path`
- `run_id` (nullable)
- `captured_at`
- `reason` (`pre_promote|pre_rollback|manual`)
- `sha256`
- `content`

## CLI Spec

### `editor review`
- Args:
  - `--run-id` (required)
  - `--json` (optional)
- Output:
  - summary table
  - per-note diff preview
  - trigger refs

### `editor promote`
- Args:
  - `--run-id` (required)
  - `--json` (optional)
- Output:
  - promoted note paths
  - missing staged paths (if any)
  - snapshot IDs created

### `editor rollback`
- Args:
  - `--note` (required; project-relative or absolute path)
  - `--snapshot-id` (optional; defaults to latest snapshot)
  - `--json` (optional)
- Output:
  - restored snapshot info
  - pre-rollback snapshot info

## Run Report Contract
- Extend run report JSON with:
  - `staged_notes[]`
  - `approved_notes[]`
  - `rolled_back_notes[]`
- Keep existing fields unchanged.

## Acceptance Criteria
- With staging enabled, v1/v2 produce staged notes and do not modify live notes.
- `editor review` surfaces file diffs and trigger refs for staged notes.
- `editor promote` atomically applies staged notes to live notes and captures snapshots.
- `editor rollback` restores prior content deterministically.
- Existing non-staging flow remains backward-compatible when `v13.enabled=false`.

## Tests
- Unit: staging path resolution and diff generation.
- Unit: promote flow creates snapshots and applies staged content.
- Unit: rollback restores selected snapshot and snapshots current content first.
- Regression: existing pipeline smoke still passes with staging disabled.

## Out Of Scope (v14)
- Multi-user approval roles.
- Web UI for review.
- Comment threads/annotations on diffs.

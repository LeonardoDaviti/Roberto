# Roberto v14 Roadmap

## Objective
Make retrieval instantaneous and ergonomic: find any claim/story/entity in under 5 seconds.

## Scope

### 1) Full-Text Search (FTS5)
- Add SQLite FTS5 indexes for:
  - tweets (`text`, `username`)
  - user notes auto blocks
  - digest/story notes auto blocks
  - idea/conflict cards
  - entities/canonical aliases
- Keep source-of-truth tables unchanged; FTS indexes are derived views.
- Add index refresh hooks in ingest/build paths.

### 2) Retrieval CLI
- Add CLI:
  - `python -m roberto_app.cli search "<query>"`
  - `python -m roberto_app.cli search "<query>" --type tweet|story|note|entity`
  - `python -m roberto_app.cli search "<query>" --json`
- Return ranked results with citations and direct file/tweet references.

### 3) Lenses (Saved Queries)
- Config file: `config/lenses.yaml`.
- Lens definition:
  - `name`
  - `query`
  - optional filters (`users`, `tags`, `entities`, `days`)
- CLI:
  - `python -m roberto_app.cli lens list`
  - `python -m roberto_app.cli lens run <name> [--json]`

### 4) Delta Timelines
- Per story/entity timeline command enhancements:
  - include `changed_since_run_id` filter.
  - include grouped change events (new evidence, confidence delta, conflict delta).

## Data Model
- Add FTS virtual tables and rebuild scripts.
- Add optional `lenses` table (or resolve directly from YAML) for run caching.

## Acceptance Criteria
- Search query returns top relevant records with citations in <1s on local cache.
- Lenses execute deterministically and produce ranked story+evidence views.
- Entity/story timeline can answer “what changed since last run”.

## Tests
- FTS index creation and refresh tests.
- Query ranking determinism on fixture.
- Lens parsing + filtering tests.
- Timeline delta tests.

## Out Of Scope (v15)
- Semantic vector re-ranker in primary path.
- Hosted search backend.
